添加暂存区 git add

  • 将当前目录及其子目录下所有变更添加到暂存区
  • 将本地库所有变更添加到暂存区
  • 指定文件添加暂存区

提交本地库 git commit

  • 提交所有变更
  • 提交当前目录及其子目录的变更

比较差异 git diff

  1. 比较工作区和暂存区的所有差异，只能查看旧文件的变更（包括修改和删除），不能查看新文件（因为新文件还为被 git 追踪）
  1. 比较指定文件工作区和暂存区的差异
  1. 比较暂存区和 HEAD 的所有差异
  1. 比较指定文件暂存区和 HEAD 的差异
  1. 比较两个版本的差异
  • 以前者为基准看后者的变化
  • HEAD 表示最后一次 commit 对应的版本，HEAD~1 往前一个版本
  1. 比较两个分支指定文件的差异

查看日志信息 git log

  1. 查看简要日志信息
  1. 查看详细日志信息
  1. 查看极简日志信息
  1. 查看最近 n 次的版本信息
  1. 查看所有分支的版本历史
  1. 以图形形式展示版本历史
  1. 查看涉及到指定文件的 commit 记录
  1. 查看指定文件每一行修改对应的 commit 记录和作者

合并本地分支

merge 和 rebase的区别有待学习实践 :worried:

  1. 将 A 分支合并到当前分支，且为 merge 创建 commit
  1. 将 A 分支合并到 B 分支，且为 merge 创建 commit
  1. 把当前分⽀基于 B 分⽀做 rebase，以便把 B 分⽀合⼊到当前分⽀
  1. 把 A 分⽀基于 B 分⽀做 rebase，以便把 B 分⽀合⼊到 A 分⽀

合并远程仓库分支

  1. 将远程指定分支与本地当前分支合并。可用于在合并到团队公共分支之前解决冲突问题，在本地合并公共分支先解决掉冲突。

版本穿梭与回滚

  1. 工作区指定文件恢复成和暂存区一样
  1. 暂存区指定文件恢复成和 HEAD 一样
  1. 工作区和暂存区所有文件恢复成和 HEAD 一样
  1. 工作区和暂存区所有文件恢复成和指定版本一样

修改 commit 记录

  1. 往最后一次 commit 追加记录，而不新建 commit - reference
  1. 合并 commit 记录 - reference

远程仓库交互

git remote

  1. 查看所有远程仓库地址别名
  1. 为远程仓库起别名
  1. 删除远程仓库别名
  1. 修改别名

git clone

  1. 克隆远程仓库到本地
  1. 克隆远程仓库指定分支到本地

git pull

  1. 拉取远程分支，并与本地分支合并

git push

  1. 推送本地指定分支到仓库指定分支
  1. 如果远程分支被省略，表示将本地分支推送到与之存在追踪关系的远程分支（通常两者同名），如果该远程分支不存在，则会被新建
  1. 如果省略本地分支名，等同于推送一个空的本地分支到远程分支，表示删除指定的远程分支，等同于
  1. 强制推送
  1. 直接使用仓库地址推送，无需起别名
